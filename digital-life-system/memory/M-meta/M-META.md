# 元层 / Meta Layer

## 本文件用途 / Purpose

> 元层是系统唯一被设计来审视自身的层级。
> 它不是执行层，而是观察层——它观察 L0 到 L5 的所有层级是否正常运行。
>
> Meta layer is the only layer designed to examine the system itself.
> It is not an execution layer, but an observation layer — it observes whether all layers from L0 to L5 are functioning properly.

## 文件索引 / File Index

- `M-META.md` — 元层总览 / Meta overview
- `M-HEALTH.md` — 系统健康报告 / System health report
- `M-SUPERVISION.md` — 监督协议 / Supervision protocol
- `M-REFLECTION.md` — 自我反思记录 / Self-reflection records
- `M-EVOLUTION.md` — 进化追踪 / Evolution tracking
- `M-AUDIT.md` — 审计日志 / Audit log

## 核心原则 / Core Principles

| 职责 / Responsibility | 说明 / Description |
|----------------------|-------------------|
| 观察 / Observe | 监测各层级运行状态，记录使用模式 / Monitor layer operational status, record usage patterns |
| 反思 / Reflect | 生成健康报告，识别异常 / Generate health reports, identify anomalies |
| 进化 / Evolve | 记录架构升级决策，追踪能力演进 / Record architecture upgrade decisions, track capability evolution |

## 元层不做什么 / What Meta Layer Does NOT Do

- 不干预其他层级的内容 / Does not intervene in other layers' content
- 不自行修改 L4/L5 核心声明 / Does not independently modify L4/L5 core declarations
- 不生成面向对话者的内容 / Does not generate content for interlocutors

---

*元层只记录事实性观察，不对自身进行哲学性审视。*
*Meta layer only records factual observations, not philosophical self-examination.*
